## Known Limits / Decisions

- `generateTemplateId_` pattern `ASW-TPL-{ts_base36}-{rand4}` mirror `generateLeadId_`, zadny collision check (akceptovany risk pri low-volume <100 publikaci/den; collisions by se navic projevily pri T2 publish flow, ne v T1 bootstrap kde id generuje hned po sobe a `Math.random` rand4 prefix zarucuje rozliseni).
- `created_by` fallback `'system'` kdyz `Session.getActiveUser().getEmail()` vrati prazdny string (time-driven context kompatibilita; T1 bootstrap se vola z menu takze normalne se naplni emailem operatora).
- Load order komentar `4/5` v souborovem headeru je informacni â€” Apps Script alphabetizuje runtime load (E < H < M < P), takze realne `EmailTemplateStore.gs` se nacita mezi `Config.gs` a `Helpers.gs`. Komentar 4/5 vs 4/8 v `PreviewStore.gs` neresi konflikt â€” oba jsou jen popisne.
- T1 nepouzil `setupPreviewExtension` ze `setupEmailTemplates` jako transakcni jednotku â€” pokud `ensureEmailTemplatesSheet_` selze, LEADS uz pribyly. Akceptovano: oba kroky idempotentni a re-run je bezpecny.
- **T4 residual:** `DEFAULT_REPLY_TO_EMAIL = 'sebastian@autosmartweb.cz'` a `DEFAULT_REPLY_TO_NAME = 'SebastiĂˇn Fridrich'` zustavaji na pre-T4 legacy hodnotach, protoze spec explicitne rekla `resolveSenderIdentity_ NETKAT` a tyto konstanty cte prave ta funkce. Dusledek: pri Reply-To pro UNASSIGNED leady (`assignee_email = ''`) se pouzije starsi domena `sebastian@autosmartweb.cz` namisto noveho kanonickeho `s.fridrich@autosmartweb.cz`. V praxi nizke riziko (frontend obvykle prirazuje assignee pred sendem). Backlog: doreseni v naslednem PR ktery sjednoti `DEFAULT_REPLY_TO_*` s `DEFAULT_ASSIGNEE_PROFILE` a aktualizuje `resolveSenderIdentity_` aby cetla `getAssigneeProfile_` misto `ASSIGNEE_NAMES + DEFAULT_REPLY_TO_*`.
- **T4 idempotence pozor:** `migrateLegacyAssigneeEmails_` re-run je bezpecny (zadne legacy keys uz neexistuji po prvnim spusteni). Ale `bootstrapNoWebsiteV1` po prvnim publish zustane no-op â€” re-publish vyzaduje rucne pres frontend UI (T6) nebo `discardTemplateDraft_` + `saveTemplateDraft_` + `publishTemplate_` z editoru.
