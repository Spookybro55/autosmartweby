## Scope

Implementace scraper runtime pro **jeden portál (firmy.cz)**. Pro 1 `ScrapingJobInput` (A-01)
vrací pole `RawImportRow` objektů (A-02) s `raw_payload_json` ve tvaru očekávaném A-03
normalizačním kontraktem. Per-record try/catch zajišťuje, že chyba 1 záznamu neshodí
celý job. Pilot pokrývá listing fetch → detail fetch → structured-data extraction →
raw row assembly → summary metrics. Zápis do Google Sheets `_raw_import` je explicitně
mimo scope (downstream krok).

### Parsing strategie (shrnutí)

1. **Listing:** regex `href="/detail/..."` → absolutní URL sada (dedup přes `Set`).
2. **Detail — primary:** JSON-LD `@type` v `{LocalBusiness, Organization, Electrician, Plumber, HomeAndConstructionBusiness, ProfessionalService, Store}` → čte `name`, `telephone`, `email`, `url`, `taxID`/`identifier.value`, `address.{addressLocality,addressRegion}`, `contactPoint.name`, `employee[0].name`, `aggregateRating.{ratingValue,reviewCount|ratingCount}`, `sameAs[0]` (pro website fallback).
3. **Detail — fallback 1:** Open Graph (`og:title` → business_name po odstranění `| firmy.cz` suffixu, `og:url` → canonical) + named meta (`canonical`).
4. **Detail — fallback 2:** regex na stable HTML patterns (`href="tel:..."`, `href="mailto:..."`, `IČO: 12345678`, `href="https://..." ... Webové stránky`).
5. **Kategorie:** primární přes `@type` (konkrétní podtyp), fallback `BreadcrumbList` (poslední element), konečný fallback na `job.segment`.

Každé pole má vlastní `try/catch` — selhání jednoho pole neselže ostatní pole stejného záznamu.

## Known Limits

1. **firmy.cz rendering stabilita.** Parser předpokládá, že initial HTML response
   obsahuje JSON-LD bloky a strukturovaná data (SEO best practice, **validováno live
   testem 2026-04-11** — 10/10 úspěšných extrakcí). Pokud by firmy.cz v budoucnu přešlo
   na primárně JS-driven rendering, listing extractor by vrátil prázdný seznam a report
   by měl `job_status=failed`; monitoring regrese přes `summary.failed / attempted` per
   `source_job_id` bude součástí A-09. Headless browser (Playwright) je out of scope pilotu.
2. **Žádný write path do Google Sheets.** Scraper produkuje RAW_IMPORT row objekty jako
   JSON. Perzistence do `_raw_import` (append-only) je samostatný downstream krok —
   vyžaduje service account / Apps Script endpoint (viz open question A-02 otázka 3).
3. **Pouze firmy.cz.** A-01 enum obsahuje i `zivefirmy.cz`, parser pro něj neexistuje.
   Přidání je jasně ohraničená práce: nový `scripts/scraper/lib/zivefirmy-cz-parser.mjs`
   jako sibling s vlastním listing URL builderem a detail parserem.
4. **Žádné unit testy.** Project má `docs:check` pro dokumentaci, ale žádný test runner
   pro `.mjs` skripty. Fixture run + live smoke test v CLI slouží jako integration test.
   Formální unit testing (vitest / node:test) je out of scope pilotu.
5. **HTML entity decoder je minimalistický.** Pokrývá běžné entity (`&amp;`, `&lt;`,
   `&nbsp;`, numeric `&#nnn;`, `&#xNN;`) pro scraper scope, není exhaustivní.
