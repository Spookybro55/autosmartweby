## company_key Algorithm

Priority order:

| Tier | Prefix | Source | Classification |
|------|--------|--------|---------------|
| T1 | `ico:` | IČO (strict 8 digits) | Primary stable identifier |
| T2 | `dom:` | Website domain (not blocked) | Secondary stable identifier |
| T3 | `edom:` | Business email domain (not freemail, not blocked) | Secondary identifier |
| T4 | `name:` | normalized_name\|normalized_city (both required) | Heuristic signal |

### Changes vs previous

| What | Before | After | Why |
|------|--------|-------|-----|
| IČO validation | ≥5 digits | exactly 8 digits (9→8 if leading zero) | Prevent false positive ICO matches |
| Domain check | no blocked filter | BLOCKED_HOST_FRAGMENTS filter | Prevent aggregator domains as keys |
| Email domain | no blocked filter | BLOCKED_HOST_FRAGMENTS filter | Prevent aggregator email domains |
| T4 fallback | name only (city optional) | name AND city required | Prevent false positive name-only matches |
| Legal suffixes | s.r.o., a.s., v.o.s., k.s. | + SE, z.s., z.ú., družstvo, o.p.s. | Handle more Czech legal forms |
| Prague city | no normalization | Praha N / Praha - X → praha | Consistent Prague dedup across districts |
