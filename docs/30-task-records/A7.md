## Design

### Trigger Architecture

Two trigger paths, both automatic:

1. **Time-based trigger** (standalone): `autoQualifyTrigger()` auto-installed by `installProjectTriggers()` (15-min timer). Scans LEADS for rows where `lead_stage` empty AND `business_name` present AND web check done. Processes up to 20 per run.

2. **Post-web-check hook** (targeted): `runQualifyForWebCheckedLeads_(leadIds)` called from `runAutoWebCheckInner_` after A-06 writes complete. Targets only just-web-checked lead_ids. Non-fatal: qualify failure does not affect web check success.

### Eligible Row Criteria

A lead is eligible for auto-qualification when ALL of:
- `lead_stage` is empty (not yet qualified/disqualified/review)
- `business_name` is not empty
- `website_checked_at` is set OR `has_website` has a value

### Fields Written

Per lead, up to 9 fields are written:

| # | Field | Value |
|---|-------|-------|
| 1 | lead_stage | QUALIFIED / DISQUALIFIED / REVIEW |
| 2 | qualified_for_preview | TRUE / FALSE |
| 3 | qualification_reason | structured reason string |
| 4 | send_allowed | TRUE / FALSE |
| 5 | personalization_level | high / medium / basic / none |
| 6 | company_key | dedupe key |
| 7 | branch_key | branch identifier |
| 8 | preview_stage | NOT_STARTED (if qualified, first time only) |
| 9 | outreach_stage | NOT_CONTACTED (if qualified, first time only) |

### Qualification Outcomes

| Outcome | Condition | qualified_for_preview | lead_stage |
|---------|-----------|----------------------|------------|
| QUALIFIED | No/weak/unknown website + has contact | TRUE | QUALIFIED |
| DISQUALIFIED (no contact) | Missing email AND phone | FALSE | DISQUALIFIED |
| DISQUALIFIED (good website) | Has real URL, not weak | FALSE | DISQUALIFIED |
| REVIEW | Known chain or enterprise keyword | FALSE | REVIEW |
