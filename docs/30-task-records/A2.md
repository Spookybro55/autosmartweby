## Scope

Navrzeni staging vrstvy `_raw_import` jako noveho system sheetu ve stejnem SPREADSHEET_ID jako LEADS. Cilem je oddelit surovy scraper output od produkcniho LEADS sheetu a zavest explicitni ingest lifecycle (raw -> normalized -> dedupe -> imported / error). LEADS zustava source of truth pro ciste leady; `_raw_import` je source of truth pro surova vstupni data a jejich lifecycle. Kontrakt definuje 16 sloupcu, 5-stavovy status model, 4-hodnotovy decision model, invariants matici a hranici mezi stagingem a produkcnim leadem.

## Contracts Changed

Novy kontrakt: **RAW_IMPORT Row v1.0**.

- 16 sloupcu celkem: 7 immutable po insertu, 9 update-in-place.
- Immutable: `raw_import_id`, `source_job_id`, `source_portal`, `source_url`, `scraped_at`, `raw_payload_json`, `created_at`.
- Update-in-place: `normalized_status`, `normalization_error`, `duplicate_candidate`, `duplicate_of_lead_id`, `lead_id`, `import_decision`, `decision_reason`, `updated_at`, `processed_by`.
- Status model: `raw` / `normalized` / `duplicate_candidate` / `error` / `imported`. Terminal: `error`, `imported`.
- Decision model: `imported` / `rejected_error` / `rejected_duplicate` / `pending_review`, nezavisly sloupec od statusu.
- Hard duplicate (ICO/domain/email domain match) jde rovnou do `error` s `rejected_duplicate`. Status `duplicate_candidate` vyhrazen pro soft dup s `pending_review`.
- Retry model: `error` terminalni, opakovani = novy raw radek s novym `raw_import_id`.
- Row id format: `RAW-{source_job_id_hash10}-{seq6}`, deterministicky v ramci jobu.
- Zavislost na A-01: `source_job_id` je FK na ScrapingJobInput z A-01 kontraktu.

## Known Limits

- `raw_payload_json` drzi scraped data jako JSON string v jedne bunce. Google Sheets limit 50k znaku/bunku — pro pilot firmy.cz detail (~2KB) dostatecne, pro HTML offload bude potreba strategie. Odlozeno do A-04.
- Archivace starych `imported` / `error` radku neni resena. Pri >9M bunek spreadsheet dosahne limit. Pro pilot nerelevantni, pro production kriticke. Odlozeno do A-09.
- Review UI pro `pending_review` neexistuje. Pro pilot staci rucni edit bunky. Dlouhodobe B-stream (frontend).
- Zadny runtime kod jeste nebezi — A-02 je pouze kontrakt a dokumentace. Implementace (sheet bootstrap, scraper write path, normalizer read path) je v A-04 a A-03 scope.
