## Scope

Minimalni review vrstva nad existujicim derived listem "Ke kontaktovani". Operator vidi preview URL, zvoli Schvalit / Zamitnout / Zmeny a rozhodnuti se atomicky propise do LEADS vcetne transition `preview_stage`. LEADS zustava source of truth; "Ke kontaktovani" je pure working layer.

## B-06 locked decisions

1. **Review surface = list "Ke kontaktovani".** Cley existujici dashboard zustava, pridavaji se jen 2 editovatelne sloupce: **Rozhodnuti ✎** (col 12) a **Duvod revize ✎** (col 13). Layout: visible 1-13, detail 14-21, Lead ID v col 21.
2. **Review decisions = enum `APPROVE | REJECT | CHANGES_REQUESTED`** ulozene do `review_decision`. Cesky dropdown `Schvalit / Zamitnout / Zmeny` + prazdna volba.
3. **Stage transitions:**
   - `APPROVE` → `preview_stage = APPROVED`
   - `REJECT` → `preview_stage = REJECTED`
   - `CHANGES_REQUESTED` → `preview_stage = BRIEF_READY` (lead se vraci do `processPreviewQueue` eligibleStages pri pristim 15-min tiku)
4. **`REJECTED` je novy preview stage** (terminal negative; nezarazuje se do eligibleStages).
5. **Audit fields zapisovane atomicky pri kazdem rozhodnuti:** `review_decision`, `reviewed_at` (ISO), `reviewed_by` (Session email nebo ''), `preview_stage`.
6. **`send_allowed` se NEPOUZIVA jako approval signal.** `review_decision` je jediny review truth.
7. **Write-back pouziva existujici `onContactSheetEdit` + `findRowByLeadId_` Variant B.** Row number NEni primary identity. `business_name + city` secondary guard ponechan.
8. **Guard rules** (decision nesmi projit pri):
   - `preview_stage != READY_FOR_REVIEW`
   - `lead_id` chybejici
   - `dedupe_flag = TRUE`
   - `lead_stage ∈ {DISQUALIFIED, REVIEW}`
   - `outreach_stage ∈ {WON, LOST}`
   - chybejici cilove LEADS sloupce (`review_decision | reviewed_at | reviewed_by | preview_stage`)
9. **Partial-write prevention:** vsechny 4 cilove sloupce se pre-resolvuji PRED prvnim `setValue`. Pri missing column nebo guard fail: **nepise se do LEADS vubec** + operator-facing note + WARN log.

## Known Limits

- **TEST RUNTIME proof neprobehl** — vyzaduje clasp push do TEST Apps Script projektu + rucni edit v TEST Google Sheets s installable onEdit trigger aktivnim. Lokalni harness pokryva logiku, ale ne zivy Sheets+Script round-trip.
- **Dropdown tolerance vs kontrakt:** `setAllowInvalid(false)` dropdown cell rejects jine hodnoty v UI, ale `reverseHumanizeReviewDecision_` akceptuje i enum pass-through (`APPROVE` / `REJECT` / `CHANGES_REQUESTED`) pro pripadne direct API volani nebo testy. UI tuto cestu nevyuziva.
- **review_note bez guard:** col 13 pouziva plain write-back (kind `review_note`) a neprovadi B-06 state guards. Zamyšlena volba: operator muze poznamkovat kontext kdykoli, poznamka je auditni, ne stateful.
- **Reviewer identity `reviewed_by` je best-effort:** `Session.getActiveUser().getEmail()` vraci prazdny retezec v kontextu kde uzivatel dal domene explicit permission (standard onEdit) — v tom pripade `reviewed_by = ''`. Nesmi to blokovat zapis.
- **`docs/27-infrastructure-storage.md` nezaktualizovan** — B-06 neprinasi infra zmenu. Task-doc-map vyzaduje Stream B → 27, ale aktualizace by byla "nic se nezmenilo". Dokumentovano vyse v Docs Updated sekci.
- **Post-B-06 layout upgrade pathway:** existujici "Ke kontaktovani" listy s pre-B-06 layoutem (11+8 cols) se pri prvnim Refresh rebuildu automaticky prepnou na novy 13+8 layout. Column group cleanup range rozsirena v `refreshContactingSheet` na 9-21 aby smazala pre-B-06 collapsed groups.
