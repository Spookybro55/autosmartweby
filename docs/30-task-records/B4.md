5. **Scope:** B-04 NEDELA slug generaci, GAS payload update, write-back do LEADS, state machine, retries, family-specific React layouty, externi persistence.

## Endpoint contract (runtime)

- **Request:** `POST /api/preview/render`, Content-Type `application/json`, header `X-Preview-Webhook-Secret: <secret>`, body = `MinimalRenderRequest` (B-01).
- **Success (200):** `{ ok: true, preview_url, preview_version: "b04-mvp-1", preview_quality_score, preview_needs_review }`.
- **Business/validation error (400):** `{ ok: false, error: "<message>" }`. GAS `PreviewPipeline.gs:1011-1012` tyto 4xx interpretuje jako throw → `preview_stage=FAILED`.
- **Unauthorized (401):** `{ ok: false, error: "Unauthorized" }`.
- **Server misconfig (500):** `{ ok: false, error: "Server misconfigured: PREVIEW_WEBHOOK_SECRET missing" }`.

B-01 `MinimalRenderResponseError` je `{ ok: false }` minimum — nepovinny `error` string je pridan pro debug (GAS ho uklada do `preview_error`), neporusuje contract.

## Known Limits

- **Live GAS integration ceka na B-05.** GAS `PreviewPipeline.gs:973-994` payload zatim neobsahuje `preview_slug` — endpoint skonci 400 (`preview_slug is required`). B-04 endpoint je hotovy a validovany, ale produkcni webhook bude funkcni az po B-05 payload fix.
- **In-memory store = reset pri restartu.** GAS je source of truth (`preview_brief_json` v LEADS, `Config.gs:84`), re-run `processPreviewQueue` obnovi stav. Externi persistence je B-06 scope.
- **Single-instance deploy.** Multi-instance Next.js (Vercel scale-out) by sdilel mezi instancemi pouze pokud by se zavedlo externi storage (B-06).
- **`preview_version="b04-mvp-1"` je staticky.** Real versioning (hashovani briefu, timestamp, diff detekce) patri do B-06.
- **`preview_screenshot_url` neni generovano.** Screenshot pipeline = B-06.
- **`KNOWN_TEMPLATE_BASES` v `quality-score.ts`** zrcadli B-03 `BASE_TO_FAMILY` keys. Pokud B-03 prida novou base, musi se doplnit. Kompromis: neporuseni "do not modify template-family.ts" pravidla.
- **Pre-existing tsc errors** v `.next/types/*.d 2.ts` (iCloud duplicity, gitignored) — nesouvisi s B-04, ale nebyly vyreseny.
- **AGENTS.md upozornuje**, ze Next.js v tomto repu ma breaking changes od training data — pri dalsich route handler pracech cist `node_modules/next/dist/docs/`. B-04 pouziva pouze standardni Web Request/Response API (ne `NextResponse`), takze je odolne vuci drift.
