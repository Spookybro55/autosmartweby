## Scope

Pridani Google OAuth prihlasovani do CRM frontendu pomoci Google Identity Services SDK. Existujici email+heslo login zustava jako fallback. Session je HMAC-SHA256 signed cookie (timing-safe verifikace pres crypto.subtle.verify).

## Contracts Changed

Nove API routes:
- POST /api/auth/google — prijima {credential}, vraci {success, email}
- POST /api/auth/logout — vraci {success}
- GET /api/auth/me — vraci {authenticated, email, provider, name}

Session cookie: crm-session (httpOnly, secure v produkci, sameSite=lax, 7 dni)

Nove env vars:
- NEXT_PUBLIC_GOOGLE_OAUTH_CLIENT_ID (klientsky — pro GIS SDK v browseru)
- GOOGLE_OAUTH_CLIENT_ID (serverovy — pro verifikaci tokenu)
