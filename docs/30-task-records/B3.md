## Scope

Navazuje na B-01 (preview brief contract) a B-02 (preview renderer). Zavadi MVP mapping vrstvu mezi runtime `template_type` (emitovanym GAS `chooseTemplateType_`) a 4 renderovaci family: `emergency`, `community-expert`, `technical-authority`, `generic-local`.

B-03 NEMENI B-01 contract, NEMENI B-02 renderer strukturu, nepridava `template_type` do `PreviewBrief`, nepridava B-04 endpoint vrstvu ani webhook aktivaci. Renderer zustava template-agnostic; family vrstva je pripravena a testovatelna pro nasledne family-specificke layouty.

## Audit Findings

1. `chooseTemplateType_` (PreviewPipeline.gs:505-554) emituje 12 base prefixu × 4 suffixy = 48 realnych `template_type` hodnot (VERIFIED)
2. Pre-existing drift: B-01 `TemplateBase` obsahoval `instalater` a `mason`, ale GAS runtime emituje `plumber` a `construction` (VERIFIED v PreviewPipeline.gs:527, 531)
3. `template_type` neni soucasti 18-field `PreviewBrief`; je to separate field ve webhook payloadu (VERIFIED v B-01 `MinimalRenderRequest`)
4. Design briefy (design-briefs/01-03) mapuji 3 design directions na 3 strukturalni sablony, ale HTML prototyp pro `03-technical-authority` v repu neexistuje (VERIFIED)
5. B-02 `/preview/[slug]` page itertuje pouze `brief.suggested_sections` a nepouziva `template_type` — zustava template-agnostic (VERIFIED v `app/preview/[slug]/page.tsx`)

## Decisions

1. 4 MVP family (ne 5): `emergency`, `community-expert`, `technical-authority`, `generic-local`. `lifestyle` (beauty/restaurant) je candidate pro budouci split, MVP routuje do generic-local.
2. Fallback na `generic-local` pro unknown/neznamy `template_type`, null/undefined, empty, i pro legacy `instalater`/`mason` stringy.
3. Suffix neovlivnuje family — suffix signalizuje website-state a rendering hints.
4. `resolveTemplateRenderHints` vraci flags only (`contactFirst`, `needsReviewFlag`, `isDataConflict`); B-03 ZAMERNE nepredepisuje UI suppression rules (hide/show) — to patri follow-up tasku po materializaci family-specific layoutu.
5. Drift fix: `TemplateBase` v `preview-contract.ts` srovnan s GAS runtime (`plumber` misto `instalater`, `construction` misto `mason`). Alternativa (oprava GAS) zamitnuta — GAS runtime je source of truth dle `autosmartweby/CLAUDE.md`.
6. `template_type` NEPRIDANO do `PreviewBrief`. Sample loader rozsiren o `SampleBriefRecord` envelope nesouci `brief` + `template_type` separately.
7. Tests pres `node --experimental-strip-types --test` (Node 24+) — no new dev-deps, imports actual TS source.

## Known Limits

- React family-specific layouty NEEXISTUJI. Renderer zustava template-agnostic. `resolveTemplateRenderHints` vraci flags, ktere zatim nikdo nekonzumuje — je to framework pro budouci layout integraci.
- `design-html/03-technical-authority/` HTML prototyp chybi — `needsReviewFlag` true pro technical-authority family, az dokud prototyp nevznikne.
- `lifestyle` family (beauty, restaurant) zamernou volbou mergnuta do `generic-local`. Budouci split vyzaduje novy design brief + HTML prototyp.
- Tests bezi pres `node --experimental-strip-types` (Node 24+). Pokud CI bezi na starsi Node, test:b03 selze — prechod na vitest/jest pripadne v pristim tasku.
- `BASE_TO_FAMILY` je definovan inline v `template-family.ts`. Pokud by se mapping meni castym procesem, lze pozdeji extrahovat do `.json` + `import`.
- Sample loader envelope `SampleBriefRecord` je mock-only. Produkce (B-04) bude `template_type` brat z webhook payloadu per B-01 `MinimalRenderRequest`.
- Drift fix je breaking change v TS typu `TemplateBase`, ale nic v crm-frontend soucasne `TemplateBase` nepouziva mimo `TemplateType` template literal (grep overeno). Apps-script kod GAS runtime je autoritativni, takze nezmenen.
