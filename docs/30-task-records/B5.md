## Scope

Navazuje na B-01 (preview contract), B-02 (preview renderer), B-03 (template family mapping), B-04 (`POST /api/preview/render` endpoint). Uzavira CRM-side smycku: Apps Script caller splnuje B-04 contract (slug v payloadu + X-Preview-Webhook-Secret header), response se parsuje do LEADS a `preview_stage` je narovnany do operator-facing lifecycle `NOT_STARTED → BRIEF_READY → GENERATING → READY_FOR_REVIEW → APPROVED`, s `FAILED` jako retry-eligible.

## Scope rules respected

1. Apps Script / CRM-side only; B-04 endpoint code beze zmeny.
2. `PreviewBrief` shape neporuseny.
3. B-03 `template-family.ts` neporuseny.

7. `APPROVED` = manualni operator stav; auto-set Apps Scriptem se nikdy nedeje.

## B-05 MVP locked decisions

1. **Auth header:** `X-Preview-Webhook-Secret` ze Script Property `PREVIEW_WEBHOOK_SECRET` pres novy helper `getPreviewWebhookSecret_()`.
2. **Payload contract:** pridat `preview_slug` do webhook payloadu (2 call sites: `processPreviewQueue`, `runWebhookPilotTest`).
3. **Lifecycle enum:** `NOT_STARTED, BRIEF_READY, GENERATING, READY_FOR_REVIEW, APPROVED, FAILED`. Legacy `QUEUED / SENT_TO_WEBHOOK / READY / REVIEW_NEEDED` ponechany v enumu pouze pro backward-compat cteni pre-B-05 dat; nove kod je nezapisuje.
4. **Success transition:** 200+ok:true → `READY_FOR_REVIEW` bez ohledu na `preview_needs_review` flag. Flag dal zijuje v `preview_needs_review` LEADS sloupci (operator signal).
5. **Failure transition:** 200+ok:false | HTTP 4xx/5xx | network exception → `FAILED` + `preview_error` s truncated diagnostikou.
6. **Retry:** `FAILED` zustava v `eligibleStages` (`['', 'not_started', 'failed', 'review_needed', 'brief_ready']`). APPROVED a READY_FOR_REVIEW NEJSOU v `eligibleStages` — pipeline je netkne.
7. **Deployment gates:** `DRY_RUN`, `ENABLE_WEBHOOK`, `WEBHOOK_URL`, `PREVIEW_WEBHOOK_SECRET` Script Property — vse operator-set, nezmenujeme Config.gs default hodnoty.

## Known Limits

- **Live PROD rollout NENI soucasti tohoto tasku.** Clasp push do TEST → validace → clasp push do PROD je operator step po merge do main. Take je treba rucne nastavit Script Properties `PREVIEW_WEBHOOK_SECRET` (match Next.js env) a `WEBHOOK_URL`.
- **In-memory B-04 store vs. GAS retries:** pokud B-04 endpoint restartuje, slug → brief cache se ztrati. GAS re-run `FAILED` rows znovu posle brief. GAS je source of truth (B-06 doresi externi persistence).
- **APPROVED auto-transition NEEXISTUJE.** Operator stav APPROVED nastavi manualne v Google Sheets. Frontend UI button je separate task.
- **`preview_attempts` counter chybi** — zadne explicitni sledovani retry pocetu. Natural-loop retry bez limit. Pokud B-04 endpoint dlouhodobe vraci 5xx, pipeline znovu a znovu zkousi. Owner rozhodnuti — scope budouciho task.
- **Legacy `REVIEW_NEEDED` v `eligibleStages`:** zachovano pro compat. Pokud by se pre-B-05 data s REVIEW_NEEDED objevila, budou re-renderovana. V produkci takova data neexistuji (DRY_RUN=true vzdy).
- **Legacy `QUEUED / SENT_TO_WEBHOOK` nejsou v `eligibleStages`.** Pokud by se nejaky row historicky zasekl v in-flight stavu, zustava neobsluhovany. Prakticky: takove rady nepreckavy, protoze DRY_RUN=true.
- **Deployment gates** (`DRY_RUN`, `ENABLE_WEBHOOK`, `WEBHOOK_URL`) zustavaji default v Config.gs. Live run se aktivuje operator zmenou, ne B-05 code zmenou.
