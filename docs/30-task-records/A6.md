## Design

### Trigger Architecture

Two trigger paths, both automatic:

1. **Time-based trigger** (standalone): `autoWebCheckTrigger()` auto-installed by `installProjectTriggers()` in PreviewPipeline.gs (same mechanism as processPreviewQueue 15-min timer). Scans ALL LEADS for rows where website_url empty AND website_checked_at empty. Processes up to 20 per run.

2. **Post-import hook** (targeted): `runWebCheckForImportedLeads_(leadIds)` called directly from `processRawImportBatch_()` in RawImportWriter.gs after import step. Targets only just-imported lead_ids. Non-fatal: web check failure does not affect import success.

**Trigger installation flow:**
```
installProjectTriggers() [PreviewPipeline.gs:1338]
  → checks existing triggers for 'autoWebCheckTrigger' + CLOCK
  → if missing: ScriptApp.newTrigger('autoWebCheckTrigger').timeBased().everyMinutes(15).create()
  → idempotent: skips if already installed
```

**Ingest hook flow:**
```
processRawImportBatch_() [RawImportWriter.gs:118]
  → for each NEW_LEAD: collect leadsRow.lead_id into stats._importedLeadIds
  → after batch loop: if _importedLeadIds.length > 0 → runWebCheckForImportedLeads_(ids)
  → try/catch: web check failure is logged WARN, does not affect import stats
```

### LEADS Fields Written

Per lead, 5 fields are written:

| # | Field | Column | Value |
|---|-------|--------|-------|
| 1 | website_url | LEGACY_COL.WEBSITE (col 13) | Found URL or "" |
| 2 | has_website | LEGACY_COL.HAS_WEBSITE (col 20) | "yes" or "no" |
| 3 | website_check_note | Helper (auto-created) | Discovery method + search query |
| 4 | website_check_confidence | Helper (auto-created) | 0.98 (email domain) or 0.65-0.95 (search) |
| 5 | website_checked_at | Helper (auto-created) | ISO timestamp — double-run guard |

### Fail Scenarios

| Scenario | Behavior | Batch Continues? |
|----------|----------|-----------------|
| Serper HTTP 429 (rate limit) | note="ERROR: HTTP 429...", website_url="", has_website="no", checked_at=set | YES |
| Serper HTTP 504 (timeout) | note="ERROR: HTTP 504...", website_url="", has_website="no", checked_at=set | YES |
| No search results | note="NOT_FOUND | q=...", website_url="", has_website="no", checked_at=set | YES (not an error) |
| No Serper API key | Logs ERROR, returns {noApiKey: true}, 0 leads checked | N/A (aborts before loop) |
| Header mismatch | Logs ERROR, returns {headerMismatch: true}, 0 leads checked | N/A (safety abort) |
| Concurrent run | LockService blocks, returns {lockFailed: true} | N/A (deferred to next cycle) |
| Web check fail in post-import hook | Logged as WARN, import stats unaffected | YES (non-fatal) |

### Batch Rules

- Default: 20 leads per run
- Rate limit: 150ms sleep between Serper calls (inherited from LegacyWebCheck)
- 20 leads × 150ms = ~3 seconds, well within GAS 6-minute execution limit
- Configurable via `opts.batchSize`
- Skipped leads (beyond batch) will be picked up by next 15-min cycle
