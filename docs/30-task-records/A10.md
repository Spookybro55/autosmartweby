## Design

### LEADS Append (appendLeadRow_)

Uses `getHeaderResolver_()` to dynamically map field names to column positions. Builds a full-width row array, populates only columns with matching headers, appends at `lastRow + 1`. No hardcoded column indices.

### Segment Taxonomy Fix

**Root cause:** `cleanSegment_()` strips diacritics and slugifies (`instalaterství` → `instalaterstvi`), but LEADS column G (`segment`) has `VALUE_IN_RANGE` validation from `SETTINGS!A2:A11` with `allowInvalid=false`. The allowed values are Czech display labels (e.g. `instalater`, `elektrikar`), not internal slugs.

**Fix:** Added `SEGMENT_SLUG_TO_LABEL_` lookup table (15 slug→label mappings covering all 10 SETTINGS segments) and `resolveSegmentLabel_()` called during normalization after `cleanSegment_()`.
