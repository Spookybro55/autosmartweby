## Normalizer Implementation (A-03)

`normalizeRawImportRow_(rawRow)` takes a full _raw_import row (16 A-02 fields) and returns:
- `{ ok: true, leadsRow: {...} }` — 23-field LEADS-ready object
- `{ ok: false, error: string, reason: string }` — rejection with audit trail

Reject reasons per A-03 contract:
| Reason | Trigger |
|--------|---------|
| INVALID_PAYLOAD_JSON | raw_payload_json parse failure |
| MISSING_BUSINESS_NAME | business_name empty or < 2 chars |
| MISSING_CITY | city empty after trim |
| NO_CONTACT_CHANNELS | both phone and email empty after cleaning |

Cleaning rules: exact match to A-03 contract (phone CZ prefix normalization, fake contact token filtering, rating no-clamp, segment dash normalization).

## Pipeline Orchestrator

processRawImportBatch_() chains:
1. Read all rows with status=raw from _raw_import sheet
2. Build LEADS dedupe index via buildLeadsDedupeIndex_()
3. For each raw row:
   - normalizeRawImportRow_() → reject (error) or continue
   - dedupeAgainstLeads_() → HARD_DUPLICATE (error) / SOFT/REVIEW (duplicate_candidate) / NEW_LEAD (import)
4. Update _raw_import row status + decision fields
5. Return batch stats

**Known gap:** Step 5 for NEW_LEAD rows updates _raw_import to status=imported and sets lead_id, but does NOT append the actual row to LEADS sheet. This is marked TODO in the code.
