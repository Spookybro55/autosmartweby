## Trigger a Workflow

### Report unit

**1 report = 1 `source_job_id`.** `source_job_id` je atomic unit scraping prace (1 query na 1 portalu v 1 city/segment). Je to jedine ID s full upstream (`_raw_import`) a downstream (LEADS) trace.

### Trigger cesty (dual path)

1. **Post-batch hook** (automatic): na konci `processRawImportBatch_()` (po A-06 auto web check), non-fatal wrap. Sebere distinct `source_job_id` z raw rows v batch-i a pro kazdy zavola `generateIngestReportForJob(jobId)`. Zapisuje do `stats.ingestReportIds` / `stats.ingestReportError`.
2. **Manual menu** (on-demand):
   - "Ingest report → Report pro source_job_id…" → prompt s job_id → `generateIngestReportForJob()`
   - "Ingest report → Report pro vsechny joby" → `generateIngestReportsForAllJobs()` (scan distinct z _raw_import + LEADS)

### Storage

- **Sheet `_ingest_reports`** — append-only, kazdy radek = 1 report. Regenerace = novy radek (comparison "teď vs předtím" se nezahazuje).
- **JSON payload v `_asw_logs`** — `aswLog_('INFO', 'generateIngestReportForJob', summary, { payload: fullReport })` → full nested breakdown vcetne `fail_reason_breakdown_json` rozbalene.

### Fail handling

- `buildIngestReport_()` je cista funkce (vstupy: sourceJobId, rawRows, leadsRows). Neprehazuje side-effects.
- `generateIngestReportForJob()` po failuru zaloguje ERROR do `_asw_logs` a throw.
- Post-batch hook v `processRawImportBatch_()` — try/catch wrap, chyba reportu nezneplatni import success.
- `generateIngestReportsForAllJobs()` — per-job try/catch, jeden fail neshodi batch reportu.

### Derived rates

| Rate | Formula | Caveat |
|------|---------|--------|
| `normalization_success_rate` | (raw_count − error_count) / raw_count | `''` on empty job |
| `import_rate` | imported_count / raw_count | `''` on empty job |
| `duplicate_rate` | duplicate_count (strict) / raw_count | Neprichazi pending_review |
| `qualification_rate` | qualified_or_beyond_count / leads_count | Uses canonical numerator (not strict current) |
| `brief_ready_rate` | brief_ready_count / qualified_or_beyond_count | `''` pokud qual_or_beyond=0; PARTIAL pokud A-08 nebehl |
| `contact_completeness_rate` | 1 − (missing_both_count / leads_count) | VERIFIED |

### Bottleneck detection

Ctyri funnel stages s pass-rates; vybira se nejnizsi:
1. **A:normalize** = (raw_count − error_count) / raw_count
2. **B:dedupe_import** = imported_count / (raw_count − error_count)
3. **C:qualify** = qualified_or_beyond_count / leads_count
4. **D:brief_ready** = brief_ready_count / qualified_or_beyond_count

`bottleneck_stage` = stage s nejnizsi rate, POKUD lowest < 0.8. Jinak `'none'`.

### Summary status

| Status | Podminka |
|--------|----------|
| `FAILED` | raw_count=0 OR (error_count / raw_count) > 0.5 |
| `PARTIAL` | imported>0 AND (lead_stage_empty>0 OR web_checked<imported) — A-06 nebo A-07 nedobehl. NEBO qualified_or_beyond>0 AND brief_ready=0 — A-08 nedobehl/dormant. |
| `DEGRADED` | bottleneck_stage != 'none' (lowest pass-rate < 0.8) |
| `OK` | jinak |

### Fail reason breakdown (JSON)

V `fail_reason_breakdown_json` sloupci:
```json
{
  "raw_decision_reasons": { "CLEAN_INSERT": 7, "HARD_DUP_ICO": 1, "MISSING_CITY": 1, ... },
  "qualification_reasons": { "NO_CONTACT": 5, "HAS_GOOD_WEBSITE": 2, "REVIEW": 1 },
  "duplicate_or_review_count": 2
}
```
- `raw_decision_reasons` grouped by raw `decision_reason` string
- `qualification_reasons` grouped by prefix (before first `:` or `;`) pro agregaci variant typu `NO_CONTACT: chybí...`
- `duplicate_or_review_count` = duplicate_count + pending_review_count (derived helper, ne top-level sloupec)

## Known Limits

- **PARTIAL status requires re-run** — jakmile dobehnou A-06/A-07/A-08 pro vsechny leady z jobu, dalsi manual regenerace prida cisty OK/DEGRADED radek (starsi PARTIAL radek zustava pro historical trend).
- **duration_ms_approx NENI exact runtime** — zahrnuje idle time mezi scrape (scraped_at per row) a batch processing (updated_at per row). Pokud scraping a ingest bezi s delay 1h, duration_ms_approx bude 1h+ i kdyz samotny processing trva 30 sekund.
- **TEST RUNTIME proof** neprobehl (mirror situace A-07/A-08) — vyzaduje clasp push + realny data v `_raw_import` a LEADS.
- **Lock scope** — post-batch hook generuje report uvnitr `processRawImportBatch_()`. Pokud batch obsahuje N distinct source_job_ids, kazdy si nacte plny `_raw_import` + LEADS (O(N×M)). Pri malem poctu jobu per batch (typicky 1-3) je to pohoda; pri velke backfill operaci by se mohlo vyplatit cache'nout nacteni.
- **source_job_id bez dat** — pokud `generateIngestReportForJob(jobId)` dostane nepouzity job_id (napr. preklep), zaloguje WARN a vygeneruje report s raw=0 leads=0 summary_status=FAILED. Ne exception.
- **`query_label` NENI samostatne pole** — composite key `portal + segment + city + district + source_job_id` je to comparison-ready tuple.
