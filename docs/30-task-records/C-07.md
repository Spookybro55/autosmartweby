## Scope

Formalizuje ingest vrstvu mezi mailbox syncem / ESP webhookem a CS1 lifecycle stavem pro 3 primarni event families (REPLY / BOUNCE / UNSUBSCRIBE) + 2 rezervni (UNKNOWN_INBOUND / COMPLAINT). Oddeluje ctyri identity vrstvy: **raw source** (Gmail thread / DSN thread / ESP webhook / manual) → **normalized event** (`InboundEvent` tvar) → **lifecycle transition** (CS1 T20/T21/T22 nebo skip) → **review flag** (`reply_needs_manual` boolean, NE CS1 state). Definuje event schemata (18 / 15 / 14 poli), 3-tier stop rule model (follow-up / address / lead), deterministickou lifecycle mapping tabulku (8 radku + 4 invarianty), idempotency kontrakt na event-level i lifecycle-level, cross-ref graf a 3 sample lifecycle scenare.

Scope je **SPEC-only** — neimplementuje mailbox polling worker, ESP webhook HTTP handler, Gmail `List-Unsubscribe` header detekci, `_asw_inbound_events` sheet creation, reply classifier, follow-up cadence engine, operator reply-handling UI ani zapisy do `apps-script/Config.gs`. Vsechny nove artefakty jsou oznacene **PROPOSED FOR C-07** a budou materializovany implementacnim taskem. Existing `apps-script/MailboxSync.gs` (`classifyReplyType_()`, `isBounceMessage_()`, `isOooMessage_()`) je VERIFIED reference pro reuse v implementaci.

- Soft-bounce escalation (N=3 → HARD promotion) je PROPOSED heuristika, ne solid threshold. Implementacni task muze parametrizovat.
- `reply_needs_manual` review flag predpoklada B6 UI pro jeho resolution. Bez B6 operator musi flag cist primo z Google Sheets LEADS (workable interim).
- Gmail neposkytuje dedikovany event pro `List-Unsubscribe` click ze strany leadu. Implementacni task detekuje pouze **explicitni unsubscribe** (reply body "unsubscribe" intent, manual operator input). List-Unsubscribe header vlastne obsluhuje appscript redirect handler (mimo C-07 scope).
- ESP webhook (SendGrid/Mailgun event webhooks) neni v C-06 sender runtime aktivovan — aktivace je blokovana C-06 handoff task. C-07 je forward-compat, ale v interim pouziva jen Gmail mailbox sync.
- `COMPLAINT` event family je rezerva bez plneho schema — aktivace az po ESP webhook enablement. Gmail sam nehlasi complaint per-thread (jen pres Postmaster Tools hromadne).
- `unsubscribe_token_sha256` je PROPOSED pro GDPR audit (token v List-Unsubscribe URL neni PII, ale hash se uklada jako evidence ze request prosel nasim endpointem). Implementacni task rozhodne o storage strategii.
- Retention policy pro `_asw_inbound_events`: append-only, immutable row-level, retention minimum 90 dni (GDPR). Konkretni TTL + archivacni strategie je mimo C-07 scope.
- PII boundary: `excerpt_plain` max 512 chars (reply/bounce) nebo 128 chars + PII-masked (unsubscribe). Plny body message se NEULOZI. Rozpor s auditem: pokud GDPR vyzaduje full raw preservation, je to separate `_asw_raw_inbound` append-only sheet (budouci task).
- Multi-event-per-message (lead posle email, ktery obsahuje reply text + "unsubscribe" intent + prichazi na ni bounce DSN soucasne) je teoreticky case — C-07 reseni "UNSUBSCRIBE > COMPLAINT > BOUNCE > REPLY > UNKNOWN" priority, ale real-world testing je implementacni task.
- Lifecycle transition guard `UNSUBSCRIBED→REPLIED` block znamena, ze pokud unsubscribed lead potom explicitne napise "chci se znovu prihlasit", C-07 to neresi automaticky — manual operator akce. Formal re-subscription flow je mimo C-07 scope.
