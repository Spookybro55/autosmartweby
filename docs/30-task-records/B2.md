## Scope

MVP preview renderer nad B-01 contractem. Vykresli route `/preview/[slug]`, ktera nacte hardcoded sample brief a vyrenderuje landing page pro remeslnika. Pouziva `PreviewBrief` a `SectionId` z B-01 bez redefinice.

### Route intent vs MVP implementace

- **Cilovy intent:** `/[slug]`
- **B-02 MVP implementace:** `/preview/[slug]`
- **Duvod:** Docasna implementacni ochrana. Root `[slug]` by kolidoval s existujicimi CRM routes (`/dashboard`, `/leads`, atd.) v soucasne single-app architekture. Prefix `/preview/` umoznuje bezpecny middleware bypass, AppShell bypass a izolaci preview layoutu. Toto neni finalni produktove rozhodnuti.

## Rendering Rules

1. Sekce se renderuji vyhradne podle `brief.suggested_sections` v poradi, jak jsou v poli
2. Renderer nerozhoduje, zda sekci zobrazit ‚Äî to rozhodl GAS pri generovani briefu
3. Uvnitr sekce mohou byt jednoduche fallbacky pro prazdna pole
4. Neznamy `SectionId` = skip

## Fallback Behavior

| Pole | Fallback |
|------|----------|
| subheadline | hero zobrazi jen headline |
| key_benefits | services zobrazi genericky text |
| cta | fallback "Kontaktujte nas" |
| contact_phone/email/name | zobrazi jen dostupne |
| rating/reviews_count | placeholder text |
| area | location zobrazi jen city |
| pain_point | faq zobrazi genericky placeholder |
